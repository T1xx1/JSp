JS+ is a TypeScript superset. By default all TypeScript syntax is valid in JS+, expect for the following exceptions:

## Breaking changes

- CommonJS is not supported. Everything is a ESM (everything is strict by default)
- `document.all` is not allowed ([MDN](https://developer.mozilla.org/en-US/docs/Web/API/Document/all))
- `typeof null -> 'null'` ([Typeof null operator](/language/runtime/typeofnulloperator))
- Negative array subscripts actually index array from the end ([Negative array subscript](/language/runtime/negativearraysubscript))
- Chained comparisons work out of the box ([Chained comparisons](/language/runtime/chainedcomparisons))

## Errors/Warnings

These errors/warnings won't block you from compiling JS+ into TypeScript/JavaScript

- `undefined` should not be used explicitly. For nullish initializations use `null` instead
- `-0`, `0/0` don't make sense
- Don't implicitly coerce strings with `+`. Use `parseInt()` or `parseFloat()` instead
- `isNaN()` is a deprecated artifact. Use `Number.isNaN()` instead
- `Date` is a deprecated artifact. Use `Temporal` instead
- Errors/warnings related to better syntax implementations ([Features](/language/features))
	:::tip[Example]
	- Prefer `do` expressions over ternaries for variable declarations ([Do expressions](/language/syntax/doexpressions))
	- Prefer `void` discard binding over underscored identifiers ([Discard binding](/language/syntax/discardbinding))
	:::
- Don't use JavaScript `#` private modifier. Prefer TypeScript `private` keyword instead
- `@ts-check` and `@ts-check` are not allowed. JS+ is typechecked by default and typechecking cannot be turned off
- `@ts-ignore` is a deprecated artifact. Use `@ts-expect-error` instead
