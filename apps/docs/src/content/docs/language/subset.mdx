JS+ is a TypeScript superset, but with stricter harnesses regarding certain ambiguous JavaScript behaviours (including TypeScript type errors).

:::caution
These harnesses won't block you from compiling JS+ to TypeScript or JavaScript.
:::

## Errors

### `+` number coercion
Prefer using `parseInt()` and `parseFloat()` instead of `+` for string-number coercion.

### `document.all` is deprecated
The `document.all` property is deprecated, and has strange behaviours harcoded into JavaScript.

### Chained comparisons
[Read more here](/language/syntax/chainedcomparisons).

### Mathematical sins
`-0`, `0/0` should not be valid things.

### Negative array subscript
[Read more here](/language/syntax/negativearraysubscript).

### `typeof null` operator
[Read more here](/language/runtime/typeofnulloperator).

### `undefined`
`undefined` by definition should be defined nor used explicitly.
